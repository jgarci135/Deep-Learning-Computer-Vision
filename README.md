## Classification of Medical image files
## Company interest project

**Description of data:**
The images were obtained from video of medical procedures.  Due to the nature of the images and the current development within the company the images and the specific objective cannot be shared and is protected by an NDA.  There is an interest in specific features that occur at a specific time point.  The images collected are roughly collected from the same time point and have 5 different classifications present.  The classifications will be labeled simply 0,1,2,3,4+ or labeled with a PN prefix.  This is a limited data set as there is only 3732 samples present.  

**Objective:**
The objective of this project is to accurately identify the 5 classifications with good accuracy.  This data set is a good candidate for use with transfer learning.  There are not enough samples for a convolution neural network to properly learn the defining features and will benefit from the use of models that have been trained on much larger data sets.

## Local Project Workflow

This repository has been split into a local, script-based pipeline so you can run the project outside of Google Colab.

### Project structure

- `data/raw/`: Put original images here in class folders (`data/raw/<class_name>/*.jpg`).
- `data/processed/`: Auto-generated train/val data after preprocessing.
- `scripts/data_preprocessing.py`: Builds train/val split and processes images.
- `scripts/model_training.py`: Trains MobileNetV2 transfer-learning classifier.
- `scripts/evaluation.py`: Generates confusion matrix and classification metrics.
- `scripts/check_class_balance.py`: Reports class imbalance and suggested weights.
- `scripts/eda_visualization.py`: Produces notebook-style exploratory plots.
- `config/project_config.json`: Central project configuration for all scripts.
- `models/`: Saved models, history, class indices, and evaluation metrics.
- `Figures/`: Training curves and confusion matrix plots.

### Windows GPU note

On native Windows, TensorFlow 2.11+ does not use NVIDIA CUDA GPUs directly.

If you want GPU acceleration, use one of these paths:

- Recommended: WSL2 + Ubuntu + NVIDIA CUDA support, then run this project in WSL.
- Alternative: TensorFlow DirectML plugin on Windows (hardware-dependent and typically less predictable than CUDA in WSL).

This repo still runs on CPU-only mode as-is, which is useful for smoke tests and smaller experiments.

Full setup steps are in [docs/wsl2_gpu_setup.md](docs/wsl2_gpu_setup.md).

### 2) Edit project config (recommended)

Set paths and hyperparameters once in `config/project_config.json`.

This lets you run scripts with minimal CLI flags.

### 3) Prepare data

```bash
python scripts/data_preprocessing.py
```

This writes:

- processed images to `data/processed/train` and `data/processed/val`
- split metadata to `splits.json`

### 4) Check class imbalance

```bash
python scripts/check_class_balance.py
```

This writes:

- class distribution plot: `Figures/class_distribution.png`
- class balance report: `models/class_balance_report.json`

### 5) Generate EDA figures

```bash
python scripts/eda_visualization.py
```

This writes:

- distribution plot: `Figures/eda_class_distribution.png`
- sample image grid: `Figures/eda_sample_grid.png`
- intensity histogram plot: `Figures/eda_histograms.png`

### 6) Train model

```bash
python scripts/model_training.py
```

This writes:

- final model: `models/trained_model.h5`
- best checkpoint: `models/best_model.keras`
- training history: `models/training_history.json`
- class mapping: `models/class_indices.json`
- plot: `Figures/training_curves.png`

### 7) Evaluate model

```bash
python scripts/evaluation.py
```

This writes:

- confusion matrix plot: `Figures/confusion_matrix.png`
- metrics JSON: `models/evaluation_metrics.json`

## Results
Using a smaller data set has proven challenging and has shown the limitations of some of the models and methods avaible for use.  There is more work to be done with this data set as the imbalance of the classes has proven the main issue when looking for the objective of high and correct calssification accuracy.

* Use different models
* Augment the minoirty data samples to balance the classes of the data
* The use of oversampling/undersampling may be useful
* Generate synthetic data

I believe there can be success in correctly classifying this data set with transfer learning. Some modifications are needed.
